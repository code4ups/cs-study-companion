    # B4 – Αφηρημένοι Τύποι Δεδομένων (ADTs)
    <GuidingQuestion lang="el">
    Ποιοι αφηρημένοι τύποι δεδομένων (ADTs) είναι καταλληλότεροι για διαφορετικά προβλήματα;
    </GuidingQuestion>

    Ένας αφηρημένος τύπος δεδομένων (ADT) είναι μια έννοια της επιστήμης υπολογιστών που χαρακτηρίζει έναν τύπο δεδομένων με βάση τη λειτουργικότητά του και όχι με βάση την εσωτερική του δομή. Αυτό σημαίνει ότι δεν μας ενδιαφέρει πώς αποθηκεύονται τα δεδομένα εσωτερικά ή πώς εκτελούνται οι λειτουργίες, αλλά τι μπορεί να κάνει ο χρήστης με αυτά τα δεδομένα.

    Για παράδειγμα, σε ένα βιντεοπαιχνίδι μπορεί να υπάρχει ένα «inventory» (αποθήκη αντικειμένων), όπου αποθηκεύεις όπλα, φίλτρα ή υλικά κατασκευής. Σε αυτό το inventory μπορείς να προσθέτεις νέα αντικείμενα, να αφαιρείς όσα δεν χρειάζεσαι ή να ελέγχεις αν έχεις κάποιο συγκεκριμένο αντικείμενο και σε ποια ποσότητα.

    Ως παίκτης, δεν χρειάζεται να γνωρίζεις πώς το πρόγραμμα κρατάει τον έλεγχο όλων αυτών των αντικειμένων. Μπορεί να διαχειρίζονται με λίστα, πίνακα ή με οποιονδήποτε άλλο τρόπο επέλεξαν οι δημιουργοί του παιχνιδιού. Αυτό που έχει σημασία είναι ότι μπορείς να δουλεύεις με το inventory (προσθήκη, αφαίρεση, έλεγχος) χωρίς να σε απασχολεί ο τεχνικός τρόπος που γίνεται αυτό στο παρασκήνιο.

    Έτσι ακριβώς λειτουργεί και ένας ADT. Σου δίνει τους κανόνες για το πώς μπορείς να χρησιμοποιείς και να διαχειρίζεσαι τα δεδομένα (π.χ. προσθήκη ή αφαίρεση πληροφορίας), αλλά όχι τις λεπτομέρειες του πώς εκτελούνται οι λειτουργίες αυτές. Ονομάζεται «αφηρημένος» γιατί αποκρύπτει την πολυπλοκότητα της αποθήκευσης και της διαχείρισης.

    Όταν λοιπόν οι επιστήμονες υπολογιστών μιλούν για ADTs, εννοούν τη δημιουργία ενός πλαισίου που ορίζει ποιες ενέργειες μπορούν να γίνουν με τα δεδομένα, χωρίς να χρειάζεται να γνωρίζουμε τις λεπτομέρειες της αποθήκευσης. Αυτό βοηθά τους προγραμματιστές να επικεντρώνονται στην επίλυση του πραγματικού προβλήματος (όπως η σχεδίαση ενός inventory) και όχι στον μηχανισμό οργάνωσης των δεδομένων στο παρασκήνιο.
